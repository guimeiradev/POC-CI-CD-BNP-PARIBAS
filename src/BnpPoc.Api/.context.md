The ASP.NET Core Minimal API application built by this PoC — a health-check
endpoint plus a Dapper/SQL Server-backed deployment-record data access
layer, used as the artifact that flows through the Jenkins CI pipeline to
Nexus.

## Summary

- BnpPoc.Api.csproj — `Microsoft.NET.Sdk.Web` project targeting `net10.0`, with `Nullable` and `ImplicitUsings` enabled. References `Dapper` (2.1.79) and `Microsoft.Data.SqlClient` (7.0.2).
- appsettings.json — carries `ConnectionStrings:BnpPocDb`, defaulting to `localhost,1433` for host-machine dev/test; overridden inside the Jenkins container via the `ConnectionStrings__BnpPocDb` environment variable set in `cicd-poc/docker-compose.yml`.
- Data/SqlConnectionFactory.cs — `IDbConnectionFactory` interface + `SqlConnectionFactory` implementation. Registered as a DI **Singleton**, but `Create()` returns a brand-new `SqlConnection` on every call — never a shared live connection. This must be preserved exactly; inverting it (holding one live connection in the container) would corrupt state across concurrent requests.
- Deployments/DeploymentRecord.cs — `DeploymentRecord` (read model) and `CreateDeploymentRequest` (write model) records.
- Program.cs — Minimal API entry point; reads `ConnectionStrings:BnpPocDb` and throws `InvalidOperationException` at startup if missing (fail-fast); registers `IDbConnectionFactory` in DI; exposes `GET /health` (unchanged, returns `{"status":"healthy"}` as `200 OK`), `POST /deployments` (validates required fields, inserts via Dapper `ExecuteScalarAsync<int>`, returns `201 Created`), and `GET /deployments` (Dapper `QueryAsync<DeploymentRecord>`, most recent first); exposes `public partial class Program {}` so `WebApplicationFactory<Program>` in the test project can bootstrap it in-process.

## Constraints

- The `Program` class MUST remain `public partial` — `BnpPoc.Api.Tests/HealthEndpointTests.cs` (and the Phase 4 `DeploymentEndpointsTests.cs`) depend on it being accessible via `WebApplicationFactory<Program>`.
- `IDbConnectionFactory.Create()` MUST keep returning a fresh `SqlConnection` per call — do not simplify to a cached/shared connection.
- The `deployment_record` table (created by the Liquibase changelog under `db/`) must exist for `POST /deployments` / `GET /deployments` to work at runtime; the code compiles regardless.
