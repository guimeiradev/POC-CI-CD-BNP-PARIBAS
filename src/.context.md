## Summary

- CicdPoc.sln — Visual Studio solution referencing `CicdPoc.Api` and `CicdPoc.Api.Tests`; both are restored/built/tested via `dotnet <cmd> src/CicdPoc.sln` in the Jenkins pipeline.
- CicdPoc.Api/ — the ASP.NET Core Minimal API application (the artifact that gets published and uploaded to Nexus).
- CicdPoc.Api.Tests/ — xUnit integration tests for `CicdPoc.Api`.

## Constraints

- The solution file's project names and GUIDs MUST stay in sync with the two `.csproj` files it references — the Jenkinsfile builds/tests via the `.sln`, not individual projects.
