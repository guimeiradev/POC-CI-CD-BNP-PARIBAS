Root of a CI/CD proof-of-concept for BNP Paribas: a minimal .NET API used as the
build target, a Jenkins declarative pipeline that builds/tests/publishes it, and
a `cicd-poc/` Docker Compose stack that hosts the pipeline's supporting services.

## Summary

- Jenkinsfile — declarative pipeline (10 stages: Restore → Database Migration → SonarQube Analysis → SonarQube Quality Gate → OWASP Dependency-Check → Semgrep → Publish → Archive → Upload to Nexus → Deploy) run by the Jenkins job configured against this repo; auto-triggered via `pollSCM` and consuming the `bnp-shared` Shared Library.
- README.md — project overview, tool-by-tool rationale, architecture diagram, phase roadmap (Fase 1–6), and run instructions.
- .gitignore — excludes `.agents/`, `.memory/`, `opencode.json`, `.ignore` from version control.
- cicd-poc/ — Docker Compose stack (Jenkins, SonarQube, Postgres, Nexus, Vault) plus its setup docs.
- src/ — the .NET solution built by the pipeline (API + tests).

## Constraints

- The Jenkins job's Script Path MUST remain `Jenkinsfile` at repo root (per `cicd-poc/README.md` job setup instructions) — moving it breaks the configured Jenkins Pipeline job.
